# Инструкция: vjoy_feeder.py — коррекция осей через vJoy

> 🔄 **vjoy_feeder.py** — фоновый «фидер»: читает физический джойстик через `winmm`, применяет коррекцию осей из профиля `joydiag_profile_final.json` и отдаёт исправленные значения на виртуальное устройство **vJoy**. Переживает переподключение джоя и перезагрузку ПК.

## Зачем он нужен

`joy_diag.py` только строит профиль. Чтобы коррекция работала **постоянно** (в игре, без открытого GUI), работает `vjoy_feeder.py`: он непрерывно передаёт скорректированные оси на vJoy, а игра (MSFS) видит только ровное vJoy-устройство.

## Требования

- **Windows** + **Python 3.8+**.
- Установленный **vJoy** с включённым устройством #1 (через *vJoyConf*) и библиотека `vJoyInterface.dll` (фидер ищет её в системе; если не находит — положи DLL рядом со скриптом).
- Рядом со скриптом — файл `joydiag_profile_final.json` (если его нет, фидер использует встроенный профиль по умолчанию).
- Файл в кодировке **UTF-8** (не пересохраняй как ANSI).

## Быстрый запуск

```
python vjoy_feeder.py
```

Фидер найдёт физический джой по VID/PID из профиля, захватит vJoy-устройство и начнёт передавать оси. Всё пишется в `vjoy_feeder.log` рядом со скриптом.

> 🪟 Чтобы работало без чёрного окна консоли, запускай через `pythonw.exe vjoy_feeder.py` (именно так делает автозапуск).

## Правильная последовательность внедрения

1. В *vJoyConf* включи vJoy-устройство #1 (нужные оси: X, Y, Z, Rx/Ry/Rz и т.д.).
2. (Опционально) в `joy_diag.py` построй профиль и сохрани `joydiag_profile_final.json` рядом с фидером.
3. Запусти `vjoy_feeder.py` и проверь в *Параметрах игровых устройств*, что vJoy-оси двигаются ровно.
4. Скрой физический джой через **HidHide** (`setup_hidhide.ps1`, от админа) — чтобы в MSFS не было дублирования осей.
5. Поставь фидер в автозапуск (`setup_autostart.ps1`).

> ↩️ **Откат:** `setup_hidhide.ps1 -Off` (вернёт видимость физического джоя) и `setup_autostart.ps1 -Remove` (уберёт автозапуск).

## Автоцентровка (фикс «иногда уводит»)

При каждом подключении джоя (и при старте/перезагрузке) фидер около 1.2с слушает покой и заново вычисляет реальный центр стик-осей. Это лечит симптом «после переподключения то ок, то нет» — центр больше не фиксированный из профиля.

> 🛡️ **Защиты** (чтобы автоцентр не «запомнил» неверный центр): если в момент замера стик трогают (разброс > 6%) или центр подозрительно далёкий (> 0.6) — фидер оставляет значение из профиля. Газ/слайдер никогда не центрируются.

### Флаги командной строки

| Флаг | Что делает |
| --- | --- |
| `--no-autocenter` | Отключить автоцентровку (брать центр только из профиля) |
| `--autocenter-secs 1.2` | Длительность замера центра в секундах |

Пример: `python vjoy_feeder.py --autocenter-secs 2.0`

## Устойчивость к сбоям

- **Переподключение джоя** — фидер ждёт и снова находит устройство по VID/PID, перезахватывает vJoy, делает автоцентровку.
- **Потеря vJoy** — повторный захват устройства.
- **Неожиданная ошибка** — внешний цикл перезапускает фидер через 3с.
- POV передаётся напрямую без изменений.

## Автозапуск (`setup_autostart.ps1`)

Регистрирует задачу в Планировщике заданий: запуск `pythonw.exe` при входе в систему, с правами админа, с авто-перезапуском при падении и задержкой 15с после логина.

```
powershell -ExecutionPolicy Bypass -File setup_autostart.ps1
```

- `-Remove` — удалить задачу автозапуска.
- `-TaskName vJoyFeeder` — имя задачи (по умолчанию).
- `-ScriptDir <путь>` — папка со скриптом.

## Скрытие физического джоя (`setup_hidhide.ps1`)

Для устранения дублирования осей в MSFS нужно скрыть реальный джой от игр, оставив только vJoy. Скрипт использует `HidHideCLI.exe` (установи HidHide с github.com/nefarius/HidHide/releases).

```
powershell -ExecutionPolicy Bypass -File setup_hidhide.ps1
```

- `-List` — показать устройства и пути.
- `-Off` — отключить скрытие (вернуть видимость джоя).
- `-DevicePath <путь>` — явно указать устройство (содержит `VID_07B5&PID_0317`).

> ⚠️ Запускай `setup_hidhide.ps1` **от администратора**. После скрытия проверь `vjoy_feeder.log`: если там «джойстик не найден» — фидер тоже перестал видеть физическое устройство (тогда нужно добавить фидер в белый список HidHide — скрипт это делает через `--app-reg`).

## Диагностика проблем

- **Оси не двигаются в vJoy** — проверь, включено ли vJoy-устройство #1 и совпадает ли `vjoy_target` в профиле.
- **«Джойстик не найден» в логе** — VID/PID в профиле не совпадают с реальными, либо джой скрыт HidHide.
- **vJoy busy/занят** — другая программа уже держит устройство; закрой второй экземпляр фидера.
- **Центр всё равно уводит** — увеличь `--autocenter-secs`, либо перестрой профиль в `joy_diag.py`.
