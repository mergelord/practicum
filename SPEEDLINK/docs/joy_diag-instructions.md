# Инструкция: joy_diag.py — диагностика джойстика

> 🎯 **joy_diag.py** — GUI-программа для диагностики winmm-совместимых джойстиков/манипуляторов в Windows: показывает оси, кнопки, POV, VID/PID, замеряет стабильность USB-портов, дрожание, покой и края, а затем строит профиль коррекции (`joydiag_profile_final.json`) для `vjoy_feeder.py`.

## Требования

- **Windows** (программа использует системный `winmm` — на Linux/Mac не работает).
- **Python 3.8+** с модулем `tkinter` (входит в стандартный установщик Python для Windows).
- Файл сохранён в кодировке **UTF-8**.
- Для preview-коррекции в реальном времени — установленный **vJoy** с включённым устройством и `pyvjoy`.

## Совместимость

Диагностическая часть работает для устройств, которые Windows распознаёт как игровые контроллеры и показывает в `joy.cpl`.

Ограничения идут от `winmm`:

| Ограничение | Деталь |
| --- | --- |
| До 16 устройств | id 0–15 |
| До 6 осей на устройство | X, Y, Z, R, U, V |
| До 32 кнопок + 1 POV | хвост сверх лимита обрезается |
| Только game controllers | устройство должно быть видно в `joy.cpl` |
| XInput-геймпады | триггеры могут отдаваться некорректно через winmm |

## Запуск

```powershell
python joy_diag.py
```

Если двойной клик по `.py` ничего не делает или мелькает консоль — запускай из командной строки, чтобы видеть ошибки.

## Главный порядок работы: сначала порты, потом профиль

Правильная логика первичной настройки такая:

```text
1. найти стабильные USB-порты
2. на стабильных портах проверить дрожание 60с
3. выбрать 1–2 лучших порта
4. только потом на финальном порту снять покой и края
5. сохранить профиль
```

Это важно: профиль покоя/краёв имеет смысл снимать уже на выбранном стабильном порту, а не до тестов портов.

## Шаг 1. Подключить устройство и проверить видимость

1. Подключи контроллер в первый USB-порт-кандидат.
2. Запусти `python joy_diag.py`.
3. В списке **«Устройство»** выбери нужный контроллер.
4. Проверь, что в live preview двигаются оси, кнопки и POV.

## Шаг 2. Тест стабильности USB-портов

Цель — понять, какие USB-порты дают стабильный центр после переподключения.

Для каждого USB-порта-кандидата:

1. В поле **«Метка порта»** впиши понятное имя: `rear1`, `rear2`, `front1`, `hub1` и т.п.
2. Убери руки с контроллера.
3. Нажми **«Снять раунд»**.
4. Переподключи контроллер в следующий порт или повтори замер на том же порту.
5. Сделай по 2–3 раунда на каждый порт.
6. Нажми **«Рекомендация»**.

Программа сравнит центр X/Y, шум и разброс между раундами, затем выберет более стабильные порты.

Результаты пишутся в:

- `port_center_log.csv`
- `port_center_log_summary.txt`

## Шаг 3. Тест дрожания на стабильных портах

После первого отбора не нужно тестировать все порты подряд. Берём только те, которые прошли тест стабильности.

Для каждого стабильного порта:

1. Подключи контроллер в этот порт.
2. Впиши ту же метку порта.
3. Убери руки с контроллера.
4. Нажми **«Замер дрожания (60с, все оси)»**.
5. Повтори для остальных стабильных портов.
6. Нажми **«Рекомендация по дрожанию»**.

Программа ранжирует порты по средним `σ` и `p2p` по всем осям.

Результаты пишутся в:

- `port_jitter_log.csv`
- `port_jitter_log_summary.txt`

Итог этого шага — выбрать один основной порт или 1–2 лучших порта-кандидата.

## Шаг 4. На финальном порту снять покой

Только теперь, когда порт выбран:

1. Оставь контроллер в финальном порту.
2. Убери руки полностью.
3. Нажми **«Замерить покой (8с)»**.

Программа посчитает:

- средний центр;
- шум;
- σ;
- drift;
- рекомендуемую deadzone;
- тип оси (`stick` или `throttle`).

## Шаг 5. На финальном порту снять края

Нажми **«Калибровка краёв (12с)»** и в течение 12 секунд прокрути каждую ось до упора во все стороны.

Новая версия пишет подробный отчёт по каждой оси:

```text
X: min=-0.9820 max=+0.9910 span=1.9730 → сохранено
Y: min=-0.9700 max=+0.9800 span=1.9500 → сохранено
U: min=+0.0000 max=+0.0000 span=0.0000 → пропущено
```

Калибровка краёв нужна, если физическая ось не доходит до полного диапазона. Если устройство и так отдаёт полный ход, этот шаг может ничего не изменить, но он безопасен: слишком маленькое движение оси пропускается и явно показывается в журнале.

## Шаг 6. Сохранить профиль

Нажми **«Сохр. JSON»** и сохрани профиль как:

```text
joydiag_profile_final.json
```

Профиль содержит:

```json
{
  "device": {
    "name": "...",
    "vid": "0x0000",
    "pid": "0x0000",
    "joy_id": 0
  },
  "vjoy_target": 1,
  "correction": {}
}
```

Именно этот файл читает `vjoy_feeder.py`.

## Шаг 7. Проверить профиль через vJoy

После сохранения профиля запусти:

```powershell
python vjoy_feeder.py
```

Открой `joy.cpl` и проверь vJoy Device:

- self-centering оси стоят ровно в центре;
- газ/слайдер ходит по диапазону;
- кнопки работают;
- POV работает.

Только после этого имеет смысл включать HidHide и автозапуск.

## Калибровка Windows

Кнопка **«Калибровка Windows (чтение)»** читает сохранённую калибровку Windows для выбранного VID/PID.

Кнопка **«Безопасный сброс калибровки Windows»** больше не удаляет ветки реестра автоматически. Это сделано специально: в `Joystick\OEM\VID_...` могут храниться OEM-данные и имя устройства из vendor-драйвера. Удаление всей ветки может превратить устройство в generic `USB Game Controller`.

Безопасный сброс теперь делается через Windows UI:

```text
joy.cpl → физический контроллер → Свойства → Параметры → Сбросить → переподключить USB
```

Если нужно вернуть vendor-имя устройства, переустанови драйвер производителя или удали устройство в Диспетчере устройств и поставь драйвер заново.

## Связка с vjoy_feeder.py

1. В `joy_diag.py` выбери стабильный USB-порт и сними профиль.
2. Сохрани `joydiag_profile_final.json` рядом с `vjoy_feeder.py`.
3. Запусти `python vjoy_feeder.py`.
4. Если игра видит два устройства — скрой физический контроллер через HidHide, а в игре назначай только vJoy.

Если подправил профиль — просто замени `joydiag_profile_final.json` и перезапусти фидер.

## Частые ошибки

- **Снимать покой до выбора порта.** Так профиль может оказаться привязан к нестабильному подключению.
- **Тестировать дрожание на всех портах без предварительного отбора.** Это долго и обычно не нужно: сначала отсекаются нестабильные порты короткими раундами.
- **Менять порт после сохранения профиля.** Если порт меняется, желательно хотя бы проверить центр и при необходимости переснять профиль.
- **Пересохранять `.py` файлы в ANSI/cp1251.** Нужен UTF-8.
