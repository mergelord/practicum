# Инструкция: joy_diag.py — диагностика джойстика

> 🎯 **joy_diag.py** — GUI-программа для диагностики любых джойстиков/манипуляторов в Windows: показывает оси, кнопки, POV, VID/PID, замеряет покой и края, ищет причину увода осей и строит профиль коррекции (`joydiag_profile_final.json`) для `vjoy_feeder.py`.

## Требования

- **Windows** (программа использует системный `winmm` — на Linux/Mac не работает).
- **Python 3.8+** с модулем `tkinter` (входит в стандартный установщик Python для Windows).
- Файл сохранён в кодировке **UTF-8** (скачанный файл уже в UTF-8 — не пересохраняй его как ANSI/cp1251, иначе будет `SyntaxError`).
- Для коррекции в реальном времени — установленный **vJoy** с включённым устройством (по умолчанию #1) и `vJoyInterface.dll`.

## Запуск

```
python joy_diag.py
```

Если двойной клик по `.py` ничего не делает или мелькает консоль — запускай из командной строки, чтобы видеть ошибки.

## Интерфейс — обзор сверху вниз

1. **Устройство** — выпадающий список найденных манипуляторов + кнопка **«Обновить список»**.
2. **Строка характеристик** — VID/PID, число осей/кнопок, наличие POV.
3. **Оси (живой просмотр)** — по строке на каждую ось: значение и две точки на шкале (синяя = «сырое» значение, зелёный контур = после коррекции).
4. **Кнопки / POV** — какие кнопки нажаты и положение POV-шляпки.
5. **Управляющие кнопки** (строка 1): «Замерить покой», «Калибровка краёв», «Сброс калибр.».
6. **Управляющие кнопки** (строка 2): «Сохр. JSON», «Сохр. CSV», «Загрузить JSON», поле «vJoy #», «Запустить коррекцию».
7. **Диагностика железа (почему уводит)**: «Тест дрожания», «Тест переподключения», «Сброс теста», «Калибровка Windows».
8. **Отчёт** — текстовое поле снизу, куда выводятся все результаты.

## Базовый сценарий: построить профиль коррекции

### Шаг 1. Выбрать устройство

Открой программу → в списке **«Устройство»** выбери свой джойстик (например `id 0 - Microsoft PC-joystick driver [VID 07B5 / PID 0317]`). Если его нет — нажми **«Обновить список»**. Сверху появятся характеристики: число осей, кнопок, наличие POV.

### Шаг 2. Замерить покой

> ✋ Убери руки с джойстика полностью. Не касайся стика и педалей.

Нажми **«Замерить покой (8с)»**. 8 секунд программа собирает значения осей в состоянии покоя и считает для каждой оси: средний центр, разброс (шум), рекомендуемую мёртвую зону и определяет тип оси (стик / руль / газ / слайдер).

### Шаг 3. Калибровка краёв

Нажми **«Калибровка краёв (12с)»** и в течение 12 секунд **прокрути каждую ось до упора во все стороны** (стик — по кругу до краёв, газ — от минимума до максимума, руль/педали — полностью). Программа запомнит реальные минимум/максимум каждой оси.

### Шаг 4. Проверить коррекцию вживую

В поле **«vJoy #»** укажи номер своего vJoy-устройства (по умолчанию `1`) и нажми **«Запустить коррекцию»**. Подвигай стик: синяя точка — сырое значение, зелёная — после коррекции. Центр в покое должен встать ровно по середине. Кнопку можно нажать повторно, чтобы остановить коррекцию.

### Шаг 5. Сохранить профиль

Нажми **«Сохр. JSON»** → сохрани как `joydiag_profile_final.json`. Именно этот файл читает `vjoy_feeder.py`. При желании **«Сохр. CSV»** выгрузит сырые замеры для анализа в Excel.

> 🔁 **«Загрузить JSON»** — подгружает ранее сохранённый профиль обратно в программу (чтобы посмотреть/проверить/перезапустить коррекцию с теми же параметрами). **«Сброс калибр.»** — обнуляет текущие замеры, если хочешь начать заново.

## Раздел «Диагностика железа (почему уводит)»

Этот блок отвечает на вопрос «почему ось уводит и иногда после переподключения всё ок, а иногда нет». Все три теста работают **по всем осям** и учитывают тип каждой оси (стик/руль центрируются, газ/слайдер — нет).

### Тест дрожания (60с)

Убери руки и нажми **«Тест дрожания (60с)»**. Долгий замер покоя отделяет три причины. Для каждой оси выводится строка с типом `[stick/throttle/...]`, центром, σ (стандартное отклонение), шумом и дрейфом во времени, и вердикт:

| Что видно | Причина | Лечение |
| --- | --- | --- |
| Высокий ШУМ (σ/разброс большой) | Износ или грязь потенциометра | Чистка/замена потенциометра, мёртвая зона |
| Стабильный УВОД центра (шум мал) | Механика/сохранённая калибровка | Автоцентр в vjoy_feeder.py или center_offset |
| ПЛЫВЕТ во времени (дрейф) | Просадка USB-питания / прогрев | Другой USB-порт, без хаба |

Для газа/слайдера центр не оценивается (он по своей природе стоит у упора) — проверяется только шум и дрейф.

### Тест переподключения

Прямой тест симптома «после пары переподключений то ок, то нет»:

1. Убери руки → нажми **«Тест переподключения»** (снимает центр за 1.5с — это раунд 1).
2. **Физически отключи и снова подключи джойстик** (USB).
3. Снова нажми **«Тест переподключения»** — раунд 2. Повтори 3–4 раза.

Программа строит таблицу центров по раундам и считает разброс центра для каждой оси. Если у стик-оси разброс **> 5%** — она помечается как главный подозреваемый: центр ловится по-разному при каждом подключении (это именно то, что лечит автоцентр в `vjoy_feeder.py`). Абсолютные оси (газ) помечаются отдельно — у них разброс обычно от позиции рычага, а не от железа.

Кнопка **«Сброс теста»** очищает накопленные раунды.

### Калибровка Windows

Нажми **«Калибровка Windows»** — программа читает из реестра сохранённую Windows-калибровку для твоего `VID&PID`.

> 💡 Важно: `winmm` отдаёт **уже откалиброванные Windows значения**, а не сырые. Поэтому если здесь центр смещён — виновата именно сохранённая в Windows калибровка, и её надо сбросить. Если ключа в реестре нет — увод чисто железный.

Как сбросить калибровку Windows вручную: **Панель управления → Устройства и принтеры → правый клик по джойстику → «Параметры игровых устройств управления» → Свойства → вкладка «Параметры» → «Сбросить».**

## Как поставить диагноз уводу оси

1. **Тест переподключения** (3–4 раунда с реальным переподключением) → стабилен ли центр между раундами.
2. **Калибровка Windows** → есть ли смещённая сохранённая калибровка.
3. **Тест дрожания (60с)** → шум против стабильного увода.

Выводы:

- Разброс между раундами большой → норма для дешёвых стиков; **автоцентр в `vjoy_feeder.py` решает это программно**.
- В реестре смещённый центр → сбрось калибровку Windows (см. выше).
- Высокий шум на оси → почисти/замени потенциометр.
- Дрейф во времени → смени USB-порт, убери хаб.

## Связка с vjoy_feeder.py

1. В `joy_diag.py` построй и сохрани `joydiag_profile_final.json` (шаги 1–5 выше).
2. Положи этот JSON рядом с `vjoy_feeder.py`.
3. Запусти `vjoy_feeder.py` — он читает профиль и применяет коррекцию на vJoy-устройство, переживая переподключения и перезагрузку. Автоцентр подстраивает центр стика при каждом подключении.
4. В игре (MSFS) скрой физический джой через HidHide, чтобы не было дублирования осей, и назначай только vJoy.

Если подправил профиль — просто замени `joydiag_profile_final.json` и перезапусти фидер. **Новый фидер для этого не нужен.**

> ⚠️ Не пересохраняй `.py` файлы в Блокноте как ANSI — только UTF-8. Иначе кириллица в комментариях ломает запуск (`SyntaxError: utf-8 codec can't decode byte`).
