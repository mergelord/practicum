# SPEEDLINK — диагностика и коррекция осей через vJoy

> 🕹️ **Контекст проекта:** диагностика winmm-совместимых игровых контроллеров, выбор стабильного USB-порта, построение JSON-профиля коррекции и подача исправленного сигнала в vJoy.
> _Обновлено: 2026-06-05. Текущая архитектура: profile-driven, без жёсткой привязки к конкретному VID/PID в коде._

## 1. Задача

Сделать Windows-программу диагностики манипуляторов и на её основе — программную коррекцию осей контроллера, которая:

1. Определяет winmm-совместимые игровые контроллеры в системе (оси, кнопки, POV, VID/PID).
2. Помогает сначала выбрать стабильный USB-порт: короткие раунды переподключения → тест дрожания 60с на стабильных портах.
3. Только после выбора порта снимает покой и края для построения профиля.
4. По JSON-профилю применяет коррекцию осей через **vJoy + vjoy_feeder.py**.
5. Работает после переподключения контроллера и перезагрузки ПК.
6. Позволяет скрыть физический контроллер от игры через **HidHide**, чтобы игра видела только vJoy.

## 2. Поддерживаемая модель

Код не привязан к конкретному устройству. Поддерживаемая область:

- Windows;
- устройства, видимые через `joy.cpl` / `winmm` / `joyGetPosEx`;
- до 16 устройств;
- до 6 осей (`X/Y/Z/R/U/V`);
- до 32 кнопок и 1 POV;
- вывод в vJoy.

Конкретный контроллер задаётся профилем:

```json
{
  "device": {
    "name": "...",
    "vid": "0x0000",
    "pid": "0x0000",
    "joy_id": 0
  },
  "vjoy_target": 1,
  "correction": {}
}
```

`joydiag_profile_final.json` — это рабочий профиль. Его можно заменить профилем другого устройства, снятым через `joy_diag.py`.

## 4. Главный алгоритм настройки

Профиль нужно снимать не сразу. Сначала выбирается стабильный USB-порт.

```text
подключение к портам
  -> короткие раунды стабильности центра
  -> отбор стабильных портов
  -> 60-секундный тест дрожания на стабильных портах
  -> выбор 1–2 финальных портов
  -> замер покоя и краёв на финальном порту
  -> сохранение профиля
  -> проверка vJoy feeder
```

### 4.1. Тест стабильности портов

Для каждого USB-порта-кандидата:

1. Подключить контроллер.
2. Вписать метку порта.
3. Убрать руки.
4. Нажать **«Снять раунд»**.
5. Сделать по 2–3 раунда на порт.
6. Нажать **«Рекомендация»**.

Результат — список стабильных портов по центру/разбросу.

### 4.2. Тест дрожания

На выбранных стабильных портах:

1. Подключить контроллер в порт.
2. Указать ту же метку порта.
3. Нажать **«Замер дрожания (60с, все оси)»**.
4. Повторить для других стабильных портов.
5. Нажать **«Рекомендация по дрожанию»**.

Результат — один основной порт или 1–2 лучших порта.

### 4.3. Замер профиля

Только на финальном порту:

1. **«Замерить покой»**.
2. **«Калибровка краёв»**, если оси не доходят до полного диапазона.
3. **«Сохр. JSON»** как `joydiag_profile_final.json`.

## 5. Архитектура общей математики

Критичная логика вынесена в `joy_core.py`, чтобы GUI и боевой фидер не расходились:

```text
joy_core.py      = общая математика коррекции
joy_diag.py      = диагностика и preview, использует joy_core.py
vjoy_feeder.py   = боевой фидер, использует joy_core.py
```

Порядок применения профиля:

```text
raw winmm value
  -> norm_axis
  -> edge calibration remap через calibrated_min/calibrated_max
  -> center_offset/deadzone/scale/invert
  -> clamp[-1..1]
  -> vJoy 1..32768
```

Калибровка краёв не только сохраняется в профиль, но реально участвует в коррекции через `joy_core.apply_profile_axis(...)`.

## 6. Profile-driven выбор устройства

`vjoy_feeder.py` выбирает физический контроллер в таком порядке:

1. `--joy-id`, если задан;
2. `--vid` + `--pid`, если заданы;
3. `profile.device.vid` + `profile.device.pid`;
4. если VID/PID не заданы и в системе ровно один контроллер — он используется;
5. если устройств несколько и VID/PID нет — нужно запустить `--list-devices` и указать устройство явно.

Примеры:

```powershell
python vjoy_feeder.py --list-devices
python vjoy_feeder.py
python vjoy_feeder.py my_profile.json
python vjoy_feeder.py my_profile.json --vid 0x07B5 --pid 0x0317
python vjoy_feeder.py my_profile.json --joy-id 0
```

## 7. Проверка и эксплуатация

1. `python vjoy_feeder.py`.
2. `joy.cpl` → vJoy Device → проверить оси, кнопки, POV.
3. При необходимости скрыть физический контроллер через HidHide.
4. После успешной ручной проверки включить автозапуск:

```powershell
powershell -ExecutionPolicy Bypass -File setup_autostart.ps1 -RunNow
```

## 8. Известные нюансы

- **Покой/края снимаются только после выбора стабильного порта.** Иначе профиль может быть снят на плохом подключении.
- **Тест дрожания 60с делается только на портах, прошедших короткий отбор.** Это экономит время и даёт осмысленное сравнение.
- **Смена порта после сохранения профиля** требует хотя бы контрольной проверки центра; при сомнениях профиль лучше переснять.
- **HidHide ↔ winmm:** фидер читает контроллер через winmm. Если после cloak устройство не читается — проверить whitelist python/pythonw.

## 9. Откат

- Скрытие: `setup_hidhide.ps1 -Off`.
- Автозапуск: `setup_autostart.ps1 -Remove`.
